01 · CONTROL

Set the bounds. Let the fleet work.

You choose the mission and authority envelope. Vigla decomposes the work, isolates every worker, audits each submission, and interrupts you only when a bound trips.

  1. 01

    Deploy across vendors

    Mix Claude Code, Codex CLI, Antigravity, and profile-backed agents in one roster.

  2. 02

    Work in isolation

    Each worker gets a dedicated git worktree and a canonical, typed event stream.

  3. 03

    Audit before merge

    The supervisor checks scope, reversibility, risk, and quality before accepting work.

  4. 04

    Revert as a mission

    Snapshot-tagged integration makes the whole accepted mission reversible at once.

02 · EVIDENCE

Judge outcomes, not keystrokes.

Every completed mission carries the record a reviewer actually needs: integrated tests, files changed, audit dimensions, unresolved issues, residual risk, and the exact snapshot used for rollback.

03 · DIFFERENCE

Parallelism is common. Governance is not.

Vendor-native tools are excellent at running their own agents. Vigla occupies the neutral layer across them, with mission-level audit and rollback.

04 · LOCAL BUILD

The artifact comes from your machine.

Vigla does not publish maintainer-built binaries. The source-locked recipe installs frontend dependencies, builds the app, ad-hoc signs it without a personal identity, verifies the disk image, and prints its SHA-256 checksum.

Requires macOS 12+, Rust 1.95, Node 22, pnpm 10, and Xcode Command Line Tools.

LOCAL / SOURCE 
git clone https://github.com/Kilbex/Vigla.git
cd Vigla
./scripts/build.sh
